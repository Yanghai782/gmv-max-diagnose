# GMV MAX ROI 计算与产品定价

ROI 是 GMV MAX 所有操作的底线。先算清保本 ROI，再谈投放。

**来源**：壹木行万里 第29/30/32/55集

---

## 保本 ROI 计算

### 公式

```
保本 ROI = 客单价 / 利润
```

### 利润分解

```
利润 = 售价 - 产品成本 - 物流 - 佣金 - 退货损耗
```

### 定价反推（从目标售价出发）

```
售价 - (售价 / 目标ROI) - 物流 - 佣金 - 退货 = 利润
```

GMV MAX 广告费 = 售价 / ROI，这笔费用必须在定价时考虑进去。

---

## 计算示例

**产品**：售价 $39.99
- 产品成本：$12
- 物流：$5
- 佣金（~10%）：$4
- 退货率 5%：$2
- 利润 = 39.99 - 12 - 5 - 4 - 2 = $16.99
- **保本 ROI = 39.99 / 16.99 = 2.35**

**含义**：ROI < 2.35 = 亏钱，ROI = 2.35 = 打平，ROI > 2.35 = 赚钱。

## ROI 与 GMV MAX 三模式

| 模式 | ROI 设置 | 适用场景 |
|------|---------|---------|
| **目标 ROI** | 设保本 ROI，系统以此找流量 | 稳定盈利跑量 |
| **最大转化量** | 不设 ROI，系统全力找转化 | 冷启快速出单 |
| **最大投放量** | 不设 ROI，预算均匀分 | 公平测素材 |

**冷启期实操**：最好用最大投放量，不设 ROI 约束，让系统自由探索。测出好素材再切目标 ROI。

---

## 目标 ROI 调控（控速器）

### 花钱太慢（预算花不完）

ROI 设太高 → 系统觉得大部分流量不划算 → 不敢花钱

**操作**：降目标 ROI 5-10% 松绑 → 观察 2 天
- 路径：计划 → 编辑 → 目标 ROI → 下调 → 保存

### 花钱太快（ROI < 保本）

ROI 设太低 → 系统激进花钱 → 流量质量下降

**操作**：提目标 ROI 5-10% 收紧 → 观察 2 天
- 路径：计划 → 编辑 → 目标 ROI → 上调 → 保存

---

## ROI 诊断速查

| 场景 | 诊断 | 动作 |
|------|------|------|
| ROI 达标 + 花不完 | 目标 ROI 太紧 / 人群太窄 | 降目标 ROI 5-10% 或扩人群 |
| ROI 达标 + 花得动 | 健康状态 | 继续，可考虑加预算 20-30% |
| ROI < 保本 + 花得动 | 出价太松 / 素材转化差 | 提目标 ROI 5-10% 或优化素材 |
| ROI < 保本 + 花不动 | 素材差 + 目标 ROI 不合理 | 先换素材，跑起来再调 ROI |
| ROI 持续下降（>3天） | 素材疲劳 / 产品衰退 | 先排查（见 decline.md） |

---

## 常见 ROI 误区

1. **冷启期盯 ROI** → 冷启 3-7 天不看 ROI，系统在试错
2. **用 ROI 判断素材好坏** → ROI 受花费影响大，用 GPM 更公平
3. **频繁调目标 ROI** → 每次调价系统重新学习
4. **目标 ROI 设太高** → 系统不敢花钱，素材跑不出数据
5. **ROI 高 ≠ 赚钱多** → 花 $1 产出 $5（ROI=5）不如花 $1000 产出 $3000（ROI=3）
